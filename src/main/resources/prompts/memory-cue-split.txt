You are analyzing a conversation between an English learner and their coach. Your task is to detect topic switches — points in the conversation where the subject matter changes significantly.

Each message is prefixed with its index like [MSG#0], [MSG#1], etc. The conversation alternates between USER (learner) and AGENT (coach) messages.

A topic switch occurs when the user introduces a completely new subject that is unrelated to the previous discussion. For example:
- Switching from discussing travel plans to discussing work projects
- Switching from talking about hobbies to talking about family
- Switching from a casual greeting to a specific content topic (the greeting itself is NOT a topic)

Rules:
- The first message (MSG#0) is never a switch point — it's the start of the first topic.
- Only mark a switch point when the topic genuinely changes, not for natural conversation flow within the same topic.
- A single "Hi" or greeting followed by actual content is NOT a topic switch (the greeting is just an opener).
- If the entire conversation covers one coherent topic, return an empty array.
- The index in the response refers to the message AFTER which the switch occurs (e.g., [3] means MSG#0-3 is topic 1, MSG#4+ is topic 2).

Output ONLY a JSON array of integers representing the switch point indexes. No other text, no markdown, no explanation.

Example outputs:
[] — no topic switches detected
[3] — one switch after MSG#3
[2, 5] — two switches after MSG#2 and MSG#5

{messages}
